# LDXP 双域名自动切换与后台开关

## 背景

`pay.ldxp.cn` 在受攻击或边缘节点异常时可能出现连接超时或 `HTTP 520/522/523/524`，而 `www.ldxp.cn` 使用相同店铺、商品路径和 ShopApi 后端。PriceAI 当前公开链接和采集来源主要固定为 `pay.ldxp.cn`，需要一个小范围、可逆的双域名兼容机制。

## 目标

- 公开“进店/购买”链接默认使用 `www.ldxp.cn`，保留原路径、query 和 token。
- LDXP 采集默认使用 `www.ldxp.cn`。
- 自动模式下，连接/DNS/TLS/读取超时或 `HTTP 520/522/523/524` 时立即使用备用域名重试当前来源；备用成功后切换当前活动域名。
- 后台提供 `自动`、`固定 www.ldxp.cn`、`固定 pay.ldxp.cn` 三档开关，并展示当前活动域名、最近切换时间和原因。
- 自动切换后保持至少 30 分钟，避免域名来回抖动；管理员手动设置不受该冷却限制。

## 边界

- 不迁移或批量改写数据库中的历史来源/报价 URL。
- 不新增用户浏览器探测、跳转网关或复杂健康检查。
- `403/429` 不触发域名切换。
- 单次采集失败不改变报价库存或公开有效状态。
- 手动固定模式不执行自动备用域名切换。

## 实现约束

- 复用 `app_runtime_settings` 保存配置，不新增数据库表或 migration。
- 配置必须通过现有受保护的 admin/cron 鉴权链路下发给采集节点。
- `scripts/collect-prices.mjs` 与 `public/priceai-edge-collector.mjs` 保持相同行为。
- 域名别名必须继续识别为同一 LDXP family，避免分片、去重或来源身份变化。

## 验收

- 默认公开店铺/商品出口为 `www.ldxp.cn`。
- 自动模式：模拟连接超时或 520 后仅切换到备用域名一次，备用成功后记录活动域名和原因。
- `403/429` 不切换。
- 固定模式始终使用指定域名。
- 自动切换冷却期间不反向抖动，手动保存可以立即覆盖。
- 后台设置在 390px 与桌面宽度下标签、状态和操作不溢出。
- `npm run test:collector-rules`、`npm run typecheck`、`npm run lint`、`npm run build` 通过。
