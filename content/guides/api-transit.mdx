<Callout title="先说结论">
API 中转站不要只看单个倍率。更实用的比较顺序是：先看充值系数，再看 Claude / GPT 模型倍率，算出综合倍率，最后结合近 7 日稳定性、样本数、来源渠道和待核验提示判断是否值得小额试用。
</Callout>

## API 中转站是什么

API 中转站通常把上游模型能力包装成兼容 OpenAI、Anthropic 或其他协议的调用入口。用户在站内充值后，通过站点提供的 Key 或 Base URL 调用 Claude、GPT 等模型。

它解决的是支付门槛、官方 API 获取门槛、不同工具接入和临时备用线路的问题。但它也引入了新的不确定性：上游来源、号池稳定性、扣费口径、售后处理、是否混池、是否二级分销。

## 真实成本怎么算

PriceAI 使用两个核心口径：

```text
充值系数 = 站内额度换算比例 / 7.2
综合倍率 = 充值系数 * 模型倍率
```

举例：

| 站内规则 | 模型倍率 | 计算方式 | 综合倍率 |
| --- | --- | --- | --- |
| `1:1` | `0.18x` | `1 / 7.2 * 0.18` | 约 `0.025x` |
| `1:1.2` | `0.21x` | `1.2 / 7.2 * 0.21` | 约 `0.035x` |
| 未公开充值比例 | `0.16x` | 缺少充值系数 | 无法计算 |

综合倍率不是官方价格，也不是绝对成本承诺。它只是把不同站点的充值比例和模型分组倍率折到同一把尺子上，方便横向比较。

## 为什么 Claude 和 GPT 分开看

中转站的 Claude 池和 GPT 池可能来自不同上游，稳定性、限流、价格和可用时段都可能不同。把所有模型混成一个全站最低价会误导用户。

PriceAI 会分开展示：

| 维度 | 作用 |
| --- | --- |
| Claude 最低综合倍率 | 判断 Claude Code、Cursor、开发工具场景的成本参考。 |
| GPT 最低综合倍率 | 判断 GPT 系模型调用成本。 |
| 模型分组倍率区间 | 看同一站点内不同标准模型的价格差。 |
| 近 7 日稳定性 | 看这个站点最近是否有足够可用样本。 |

## 号池来源怎么理解

| 标签 | 含义 | 你该怎么用 |
| --- | --- | --- |
| 官方 API | 站点声称或展示基于官方 API 额度。 | 仍要看价格、限流和售后，不代表 PriceAI 担保。 |
| 云厂商 | 可能来自云平台模型服务。 | 关注地区、限流、模型版本和价格口径。 |
| 一手自建号池 | 站点自己维护账号或资源池。 | 看是否说明 Pro / Plus / Max / Team 等池子。 |
| 一手批发 | 站点可能有批量上游资源。 | 看稳定性样本和售后规则。 |
| 二级分销 | 站点再接别人的中转或上游。 | 可以收录，但首次充值应更谨慎。 |
| 混合渠道 | 多种上游混在一起。 | 关注某个模型是否会被切到不同质量线路。 |
| 未披露 | 上游和池子信息不足。 | 只适合小额试用或继续观望。 |

## 为什么先做价格和稳定性

完整检测报告需要持续跑样本，至少包括：

| 检测项 | 说明 |
| --- | --- |
| 真模型 | 返回模型是否和标称模型一致。 |
| 协议兼容 | 是否兼容常见 SDK、工具和流式输出。 |
| TTFT / TPS | 首 Token 时间和生成速度。 |
| 并发与限流 | 多请求下是否稳定。 |
| 错误率 | 429、423、超时、空响应等错误类型。 |

这些维度值得做，但在没有稳定样本和 API Key 检测条件前，不适合承诺实时检测。当前先把价格公式、标准模型名、站点榜、详情拆解和来源披露跑通，后续再接自动检测和历史曲线。

## 综合推荐怎么排

默认“综合推荐”不是商家信誉认证，也不是充值担保。它只使用当前公开数据中的客观维度：

| 维度 | 权重 | 计算口径 |
| --- | ---: | --- |
| 成本竞争力 | 30% | 在当前模型或筛选范围内按综合倍率的对数分位比较，避免低倍率过早全部满分。 |
| 近 60 次成功率 | 18% | 使用公开监测的最近请求结果计算置信下界，短期连续失败会更快反映到排序。 |
| 近 7 日成功率 | 12% | 使用公开监测的 7 日整体成功率和样本数计算置信下界，避免短时波动完全主导。 |
| 响应延迟 | 15% | 7 日平均请求耗时占 12%，最近请求耗时占 3%；在当前筛选范围内按对数分位比较。当前候选中至少 70% 的站点具备 30 个以上新鲜延迟样本时才启用。 |
| 缓存命中率 | 10% | 同一模型家族内先按分组去重并按 Token 样本加权；“全部”视图再对有数据的模型家族等权平均。无有效样本不加分。 |
| 模型真假检测 | 15% | 只使用可打开、可复核的公开检测报告；待检测不加分，异常不加分。 |

倍率、公开监测成功率、响应延迟和缓存命中率都跟随当前模型或模型家族筛选，不能用其他模型家族的监测表现替代。`model` 级监测可以用于具体模型，并可向上聚合到模型家族和站点总览；`family` 推断只作参考，不进入硬排名。报价侧只有参考证据、但站点存在新鲜 `station + exact` 监测时，站点总览使用站点级证据兜底。“全部”视图的响应延迟和缓存都先在各模型家族内汇总，再对有数据的家族等权平均，避免单个高样本家族完全代表整站表现。监测数据超过 7 天未更新时，不进入综合推荐，但仍可在最低倍率等客观列表中查看。这里的响应延迟是监测请求总耗时，不等同于首 Token 时间（TTFT）或 TPS 输出速度；后续采集到标准化 TTFT 后会作为独立指标重新评估。

榜单统一按公开监测数据计算，公开来源类型本身不额外加分。广告、赞助、AFF、优惠码、入驻关系、站点程序、资料完整度和主观风险标签不会进入性能评分；这些信息只做独立披露、筛选或使用提醒。

## 小额试用清单

- 不要第一次就大额充值。
- 优先选择公开价格页、售后入口、余额规则和退款规则清楚的站点。
- 如果站点标注混池、二级分销、上游未披露，只把它当作备选测试。
- 对 Claude 和 GPT 分别测试，不要用一个模型的稳定性推断另一个模型。
- 购买或充值前回原站再次核验价格和规则。
- 不要把 API Key、账号密码、Cookie 发给任何第三方比价页面。

## 用 PriceAI 怎么看

1. 先进入 API 中转站价格榜，看默认综合推荐。
2. 如果你只用 Claude，筛选 `Claude` 并看 Claude 综合倍率。
3. 如果你只用 GPT，筛选 `GPT` 并看 GPT 综合倍率。
4. 想找低价时切到按倍率排序。
5. 想找备用稳定线路时切到按稳定性排序。
6. 点进站点详情，看价格表、可用性样本、来源渠道和待核验提示。
